# R-Shiny Accessibility

Implemented baseline controls:

- Semantic headings.
- Labels for inputs.
- Keyboard-accessible controls.
- Visible focus states.
- Status communicated with text as well as colour.
- `aria-live` regions for status and errors.
- Plain-language limitations.
- Responsive form layout.
- No flashing content.

This is not a formal WCAG certification. Remaining work includes assistive-technology review, full colour contrast audit and manual keyboard walkthrough across supported browsers.
## Milestone 10 Accessibility Controls

The advanced Shiny product adds a skip-to-main-content link, main landmarks, persistent review-mode status messaging, accessible upload instructions, screen-reader status regions, visible focus styles, descriptive download labels, reduced-motion CSS, minimum target sizing, and explicit required-field labels.

This is an implementation checklist, not a formal WCAG conformance claim. Manual assistive-technology review and full contrast audit remain required before production use.
