# SAS Viya Integration

## Intended Role

SAS Viya may later be used to demonstrate model-lifecycle governance, evidence capture, or comparison with local lifecycle controls.

## Genuine Integration Path

A future `real_sas_viya` path must use genuine access, real configuration boundaries, reproducible evidence, and honest limitations. Trial-access limitations must be documented.

## Local Fallback

The repository must remain runnable without SAS Viya. A local model-lifecycle approach may be labelled `local_model_lifecycle` and must not imply real SAS Viya execution.

## Evidence Requirements

Future evidence must identify model version, approval status, validation results, promotion decision, and operator. Milestone 1 does not create model artefacts.

## Current Status

SAS Viya integration is not implemented.
