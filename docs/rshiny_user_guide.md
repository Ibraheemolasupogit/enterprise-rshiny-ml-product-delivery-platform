# R-Shiny User Guide

Start the API in one terminal with a local key of your choice:

```bash
export MODEL_API_KEY="<choose-a-local-key>"
export SERVING_REVIEW_MODE="true"
python3 -m ml_product.cli serve-model-api --registry-config config/model_registry.yaml --serving-config config/serving.yaml
```

Start Shiny in another terminal:

```bash
export MODEL_API_BASE_URL="http://127.0.0.1:8000"
export MODEL_API_KEY="<same-local-key>"
export SHINY_APP_ENV="local"
Rscript -e 'shiny::runApp("rshiny", host = "127.0.0.1", port = 3838)'
```

The status banner explains whether review-mode scoring is available. If no approved active model exists and review mode is disabled, prediction is unavailable.

Use **Load synthetic example** to populate a safe demonstration record. The probability is an estimated synthetic long-stay probability, not a certainty and not a clinical recommendation. Feedback is product feedback only; do not enter identifiers or patient details.
## Milestone 10 Pages

Use Cohort Scoring only with synthetic CSV files matching the sample template. Results are generated by the local FastAPI service in review mode and can be downloaded as a bounded CSV without raw inputs.

Use Model Performance to review committed validation and locked test evidence. Use Model Governance to review current registry status. The app does not approve, activate, deploy, monitor, or retrain models.
